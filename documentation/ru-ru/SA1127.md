## SA1127

<table>
<tr>
  <td>TypeName</td>
  <td>SA1127GenericTypeConstraintsMustBeOnOwnLine</td>
</tr>
<tr>
  <td>CheckId</td>
  <td>SA1127</td>
</tr>
<tr>
  <td>Категория</td>
  <td>Правила удобочитаемости</td>
</tr>
</table>

:memo: Это правило является новым для **StyleCop Analyzers**, и его не было в **StyleCop Classic**.

## Cause

Ограничение для обобщенного параметра типа или метода находится в той же строке, что и объявление, в файле кода C#.

## Описание правила

Нарушение этого правила происходит в том случае, если код содержит одно или несколько предложений `where`, содержащих ограничения на обобщенные типы, расположенные в той же строке, что и само объявление типа или метода.

Например, следующий код приведет к двум нарушениям этого правила:

```csharp
private void Method<T, R>() where T : class where R : class, new()
{
}
```

Следующий код не приведет ни к каким нарушениям:

```csharp
private void Method<T, R>()
    где T : класс
    где R : класс, new()
{
}
```

## Как исправить нарушения

Чтобы исправить нарушение этого правила, перенесите каждое предложение `where` в отдельную строку.

## Как подавить нарушения

```csharp
#pragma warning disable SA1127 // Ограничения обобщенных типов должны располагаться на своей собственной строке
private void Method<T>() where T : class
#pragma warning restore SA1127 // Ограничения обобщенных типов должны располагаться на своей собственной строке
{
    // тело метода...
}
```
