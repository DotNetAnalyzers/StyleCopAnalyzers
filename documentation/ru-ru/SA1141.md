:memo: Это правило является новым для **StyleCop Analyzers**, и его не было в **StyleCop Classic**.
:memo: Это правило активно только для C# 7.0 и выше.

## Причина

Используется объявление типа ValueTuple вместо предпочтительной конструкции языка - кортежа. 

## Описание правила

Используется объявление типа ValueTuple вместо предпочтительной конструкции языка - кортежа. Информацию о том, как работать с кортежами в C# 7, см. в документации по [Типы кортежей](https://docs.microsoft.com/ru-ru/dotnet/csharp/tuples).

Например, следующий код приводит к нарушению этого правила:

```csharp
ValueTuple<int, int> x; // SA1141
```

Следующий код не приведет к какому-либо нарушению:

```csharp
(int, int) x;
```

## Как исправить нарушения

Чтобы исправить нарушение этого правила, используйте в коде соответствующий тип кортежа вместо типа ValueTuple.

## Как подавить нарушения

```csharp
[SuppressMessage("StyleCop.CSharp.ReadabilityRules", "SA1141:UseTupleSyntax", Justification = "Reviewed.")]
```

```csharp
#pragma warning disable SA1141 // Использовать синтаксис кортежей
#pragma warning restore SA1141 // Использовать синтаксис кортежей
```
