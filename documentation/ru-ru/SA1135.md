## SA1135

<table>
<tr>
  <td>TypeName</td>
  <td>SA1135UsingDirectivesMustBeQualified</td>.
</tr>
<tr>
  <td>CheckId</td>
  <td>SA1135</td>
</tr>
<tr>
  <td>Категория</td>
  <td>Правила удобочитаемости</td>
</tr>
</table>

:memo: Это правило является новым для **StyleCop Analyzers**, и его не было в **StyleCop Classic**.

## Причина

Директива `using` не квалифицирована.

## Описание правила

Нарушение этого правила происходит, когда директива `using` содержится в пространстве имен и не является квалифицированной.
Обратите внимание, что определения псевдонимов классов в одном и том же пространстве имен не нуждаются в квалификации.

Например, следующий код приведет к нарушению этого правила:

```csharp
namespace System.Threading
{
    using IO;
    using Tasks;
}
```

Следующий код не приведет к каким-либо нарушениям:

```csharp
namespace System.Threading
{
    using System.IO;
    using System.Threading.Tasks;

    using T = Thread;
}
```

## Как исправить нарушения

Чтобы исправить нарушение этого правила, используйте полное квалифицированное имя пространства имен.

## Как подавить нарушения

```csharp
namespace N1.N2
{
#pragma warning disable SA1135 // Директивы using должны быть квалифицированными
    using N3;
#pragma warning restore SA1135 // Директивы using должны быть квалифицированными
}
```
