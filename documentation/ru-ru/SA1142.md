:memo: Это правило является новым для **StyleCop Analyzers**, и его не было в **StyleCop Classic**.
:memo: Это правило активно только для C# 7.0 и выше.

## Причина

На элемент кортежа была сделана ссылка по имени его метаданных, когда доступно имя элемента.

## Описание правила

На элемент кортежа была сделана ссылка по имени его метаданных, когда доступно имя элемента. Информацию о том, как работать с кортежами в C# 7, см. в документации по [Типы кортежей](https://docs.microsoft.com/ru-ru/dotnet/csharp/tuples).

Например, следующий код приведет к нарушению этого правила:

```csharp
(int valueA, int valueB) x;

var y = x.Item1; // SA1142
```

Следующий код не приведет к какому-либо нарушению:

```csharp
(int valueA, int valueB) x;

var y = x.valueA;
```

## Как исправить нарушения

Чтобы исправить нарушение этого правила, используйте соответствующее имя элемента кортежа в коде вместо имени метаданных.

## Как подавить нарушения

```csharp
[SuppressMessage("StyleCop.CSharp.ReadabilityRules", "SA1142:ReferToTupleElementsByName", Justification = "Reviewed.")]
```

```csharp
#pragma warning disable SA1142 // Обращайтесь к элементам кортежа по имени
#pragma warning restore SA1142 // Обращайтесь к элементам кортежа по имени
```
