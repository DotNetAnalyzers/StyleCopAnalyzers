:memo: Это правило является новым для **StyleCop Analyzers**, и его не было в **StyleCop Classic**.

## Причина

Выполняется приведение значения вместо использования суффикса для литеральных значений.

## Описание правила

Вместо использования суффикса числа выполняется приведение. Используйте суффикс "U" для создания 32-битного беззнакового целого числа, "L" для 64-битного целого числа, "UL" для 64-битного беззнакового целого числа, "F" для 32-битного числа с плавающей точкой, "D" для 64-битного числа с плавающей точкой и "M" для чисел decimal. Суффиксы не зависят от регистра.

Например, следующий код приводит к нарушению этого правила:

```csharp
var x = (long)1;
```

Следующий код не приводит к каким-либо нарушениям:

```csharp
var x = 1L;
```

## Как исправить нарушения

Чтобы исправить нарушение этого правила, используйте соответствующий литеральный суффикс вместо приведения значения в коде.

## Как подавить нарушения

```csharp
[SuppressMessage("StyleCop.CSharp.ReadabilityRules", "SA1139:UseLiteralsSuffixNotationInsteadOfCasting", Justification = "Reviewed.")]
```

```csharp
#pragma warning disable SA1139 // Использовать литеральную суффиксную нотацию вместо кастинга
#pragma warning restore SA1139 // Использовать буквенные обозначения суффиксов вместо приведения
```
